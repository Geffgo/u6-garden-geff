# Fiche Technique : VM Stormshield SNS EVA1 (VirtualBox)

### 1. Caractéristiques de la Machine Virtuelle

- **Nom de la VM** : `VM Stormshield SNS EVA1`
- **Système d'exploitation** : BSD (FreeBSD 64-bit)
- **Ressources Système** :
    - **Processeur (vCPU)** : 2
    - **Mémoire vive (RAM)** : 2048 Mo
    - **Stockage (HDD)** : 10 Go

### 2. Configuration Réseau (VirtualBox)

Pour permettre au pare-feu de filtrer le trafic, les interfaces doivent être configurées correctement :

- **Mode d'accès** : Accès par pont (Bridged Adapter)(Adaptateur 1), Réseau interne (Adaptateur 2 et 3)
- **Interfaces activées** : Adaptateur 1, 2 et 3
- **Chemin** : `Configuration` > `Réseau` > `Mode d'accès : Accès par pont/Réseau interne`

### 3. Accès et Authentification


> [!COMPTE]
> 
> - **Accès par défaut (Usine)** : `admin` / `admin`
> - **Interface Web (Post-config)** : `admin` / `netlab123`
> -  **Console / SSH** : `admin` / `netlab123`

### 4. Procédure de Remise à Zéro (Factory Reset)

Si tu souhaites réinitialiser le boîtier avec ta configuration spécifique :

1. **Exécuter la commande de reset** :
    
    Bash
    ```
    defaultconfig -f -r -p -c -L
    ```
1. **Configuration post-redémarrage** :
    
    - **Langue/Clavier** : Choisir `4` pour le français (FR).
    - **Nouveau mot de passe** : `netlab123`.
    - **Configuration IP (DHCP)** : Répondre `No` aux deux premières demandes (pour fixer tes propres IPs plus tard).
    - **Administration distante** : Répondre `Yes` à "Autoriser l'administration sur l'interface OUT".

### 5. Aide-mémoire des commandes utiles

- **Gestion système** :
    - `reboot` : Redémarrer proprement le pare-feu.
    - `halt` : Éteindre la machine virtuelle.
- **Diagnostic réseau** :
    - `portinfo` : Affiche l'état et la configuration de toutes les interfaces réseau.

- **Services Stormshield** :
    
    - `enatconf` : Permet de modifier la configuration réseau en ligne de commande si l'interface web est inaccessible.
        

> [!TIP]
> 
> **Conseil :** Toujours essayer de se connecter directement avec le nouveau mdp sur la VM
