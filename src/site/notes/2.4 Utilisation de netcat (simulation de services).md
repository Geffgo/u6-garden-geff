# 2.4 Utilisation de netcat (simulation de services)

### Lancer un serveur TCP sur un port arbitraire

- **Ce qu'il faut faire :** On simule un service qui "écoute" les appels entrants sur un port spécifique (ici 9999). Cela permet de vérifier que le port est ouvert sur la machine.
- **Commande :** `nc -l -p 9999`
- **Action :** Le terminal reste en attente (curseur clignotant). C'est normal : il écoute.

### Se connecter au serveur TCP depuis une autre machine

- **Ce qu'il faut faire :** Depuis un second terminal (ou une autre machine), on tente de "frapper à la porte" du port 9999 pour voir s'il répond.
- **Commande :** `nc -zv [ADRESSE_IP] 9999`
- **Résultat attendu :** `Connection to [IP] 9999 port [tcp/*] succeeded!`

### Lancer un serveur UDP

- **Ce qu'il faut faire :** On fait la même chose mais en mode **UDP** (protocole plus rapide mais sans vérification de connexion, souvent utilisé pour les logs comme Syslog).
- **Commande :** `nc -u -l -p 514`
- **Action :** Le serveur attend des paquets de données sur le port 514.

### Envoyer un message UDP

- **Ce qu'il faut faire :** On envoie une chaîne de caractères "test" vers le serveur pour vérifier que les données traversent bien le réseau.
- **Commande :** `echo "test" | nc -u [ADRESSE_IP] 514`
- **Résultat attendu :** Le mot "test" doit apparaître instantanément sur l'écran du serveur (lancé à l'étape précédente).


### Lancer un serveur HTTP rapide avec Python

- **Ce qu'il faut faire :** Python permet de transformer n'importe quel dossier en site web temporaire. C'est idéal pour tester le port 8443 sans avoir à configurer tout Apache.
- **Commande :** `python3 -m http.server 8443`
- **Résultat attendu :** `Serving HTTP on 0.0.0.0 port 8443 ...`

### Accéder au serveur Python depuis le client

- **Ce qu'il faut faire :** On utilise `curl` pour demander la page d'accueil du serveur Python.
- **Commande :** `curl http://[ADRESSE_IP]:8443`
- **Résultat attendu :** Un listing HTML des fichiers présents dans le dossier où tu as lancé la commande Python.
