# 2.4 HAProxy 
### Créer une configuration frontend/backend minimale

**Ce qu'il faut faire :** Configurer HAProxy pour recevoir les requêtes (frontend) et les envoyer vers les serveurs web (backend).

- **Commande :** `vim /etc/haproxy/haproxy.cfg`
```
global
	log /dev/log	local0

defaults
	log	global
	mode	http
	option	httplog
	option	dontlognull
        timeout connect 5000
        timeout client  50000
        timeout server  50000

frontend fe_http
	bind :80
	default_backend be_web

frontend https-in
    bind :443 ssl crt /root/haproxy.pem
    default_backend be_web

backend be_web
	balance roundrobin
	option httpchk
	server web1 192.168.1.28:80 check
	server web2 192.168.1.29:80 check

```

### AC SSL

- **Creer un fichier : `setup_ssl.sh`**

```
#!/bin/bash

# Configuration
CA_CN="Mon_Autorite_Root_CA"
SITE_CN="hugo.local"

echo "--- 1. Génération de la CA et du Certificat Site ---"
# Génération CA
openssl genrsa -out ca.key 4096
openssl req -x509 -new -nodes -key ca.key -sha256 -days 3650 -out ca.crt \
    -subj "/C=FR/ST=France/L=Amiens/O=GAND/OU=IT/CN=$CA_CN"

# Génération Site
openssl genrsa -out site.key 2048
openssl req -new -key site.key -out site.csr \
    -subj "/C=FR/ST=France/L=Amiens/O=GAND/OU=INFO/CN=$SITE_CN"

# Signature
openssl x509 -req -in site.csr -CA ca.crt -CAkey ca.key -CAcreateserial \
    -out site.crt -days 365 -sha256

echo "--- 2. Création du fichier PEM pour HAProxy ---"
# Ordre : Certificat serveur, puis sa clé, puis la CA
cat site.crt site.key ca.crt > haproxy.pem

echo "--- 3. Vérification du fichier PEM ---"
if [ -f haproxy.pem ]; then
    echo "Fichier haproxy.pem créé avec succès."
    # Vérification rapide de la structure
    grep -E "BEGIN CERTIFICATE|BEGIN RSA PRIVATE KEY|BEGIN PRIVATE KEY" haproxy.pem
else
    echo "Erreur lors de la création du fichier PEM."
fi

echo "--- 4. Test de validité pour HAProxy ---"
openssl verify -CAfile ca.crt site.crt

```

- **Rendez-le exécutable** :
    ```
    chmod +x setup_ssl.sh
    ```
- **Lancez-le** :
    ```
    ./setup_ssl.sh
    ```

>[!INFORMATION]
>**FICHIER** `haproxy.pem` : a pour chemin `/root/haproxy.pem`
>


### Installer l'AC dans le navigateur du poste client

**Ce qu'il faut faire :** Faire accepter votre AC par le client pour éviter les alertes de sécurité.

- **Méthode :** Paramètres du navigateur → Certificats → Autorités → Importer **ca.crt**.
    
- **Résultat attendu :** AC visible dans la liste des autorités.
    

### Accéder au site en HTTPS via HAProxy (Test Curl avec AC)

**Ce qu'il faut faire :** Vérifier que la connexion sécurisée fonctionne en ligne de commande.

- **Commande :** `curl --cacert ca.crt https://URL_SITE`
- **Résultat attendu :** Page web affichée, pas d'erreur TLS.

### Vérifier l'absence d'avertissement dans le navigateur

**Ce qu'il faut faire :** Confirmer visuellement la sécurité du site.

- **Méthode :** Navigateur → accéder à l'URL du site.
- **Résultat attendu :** **Cadenas** sans alerte de sécurité.

### Démarrer HAProxy avec la configuration

**Ce qu'il faut faire :** Lancer le service pour appliquer tous les changements.

- **Commande :** `systemctl restart haproxy`
- **Résultat attendu :** Service actif, pas d'erreur.

### Accéder au site en HTTPS via HAProxy (Test simple)

**Ce qu'il faut faire :** Tester l'accès rapide (en ignorant les erreurs si l'AC n'est pas installée partout).

- **Commande :** `curl -k https://IP_HAPROXY`
- **Résultat attendu :** Page web affichée.

### Vérifier la répartition (arrêter un backend)

**Ce qu'il faut faire :** S'assurer que la haute disponibilité fonctionne en cas de panne d'un serveur.

- **Commande :** `systemctl stop apache2` (sur le serveur web1).
- **Résultat attendu :** Le site répond toujours (via le serveur web2).
