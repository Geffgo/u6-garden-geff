# 2.2 Switch Cisco 2960

### Réinitialiser le switch (Remise à zéro)

**Ce qu'il faut faire :** Effacer la configuration en mémoire (NVRAM) et supprimer le fichier des VLANs pour repartir sur une base usine.
- **Commandes :** 1. `write erase` (Valider avec Entrée) 2. `delete vlan.dat` (Valider avec Entrée) 3. `reload` (Confirmer le redémarrage)
- **Résultat attendu :** Le switch redémarre et demande "Would you like to enter the initial configuration dialog? [yes/no]".

### Créer un VLAN et le nommer

**Ce qu'il faut faire :** Créer un réseau virtuel (ex: VLAN 10 pour la COMPTA).

- **Commandes :**
    ```
    conf t
    vlan 10
    name COMPTA
    exit
    ```
- **Vérification :** `show vlan brief`

### Affecter un port en mode Access

**Ce qu'il faut faire :** Brancher un ordinateur (PC) sur un port spécifique (ex: FastEthernet 0/1) et le mettre dans le VLAN 10.

- **Commandes :**
    ```
    interface fa0/1
    switchport mode access
    switchport access vlan 10
    no shutdown
    ```
- **Action :** Le port Fa0/1 appartient désormais uniquement au VLAN 10.

### Configurer un port Trunk

**Ce qu'il faut faire :** Créer une "autoroute" sur un port (ex: Gigabit 0/1) pour laisser passer tous les VLANs vers un autre switch ou un routeur.

- **Commandes :**
    ```
    interface gi0/1
    switchport mode trunk
    ```
    
- **Vérification :** `show interfaces trunk` (Le port doit apparaître dans la liste avec le statut `trunking`).
    

### Vérifier la table VLAN

**Ce qu'il faut faire :** Afficher le récapitulatif pour s'assurer que les noms et les ports sont corrects.

- **Commande :** `show vlan brief`
    
- **Résultat attendu :** Un tableau listant les VLANs (1, 10, etc.), leur nom et les interfaces qui leur sont liées.
