# 2.1 Configuration Réseau Linux 

### Changer l'adresse IP d'une VM

**Ce qu'il faut faire :** Selon la distribution (Debian ou Ubuntu), on modifie le fichier de configuration pour fixer une IP statique.

- **Sur Ubuntu (Netplan) :** Éditer `/etc/netplan/00-installer-config.yaml`.
    - _Action :_ Remplacer `dhcp4: true` par `addresses: [192.168.1.10/24]`.
    - _Appliquer :_ `sudo netplan apply`


- **Sur Debian (/etc/network/interfaces) :** Éditer le fichier.
    - _Action :_ Rajouter une interface et son IP 
    - _Appliquer :_ `systemctl restart networking`


> [!Changement]
> 
>```
># Lignes déjà présentes :
>source /etc/network/interfaces.d/*
>auto lo
>iface lo inet loopback
>
># --- CONFIGURATION À AJOUTER ---
># Remplace eth0 par le nom de ton interface (trouvé via "ip a")
>auto eth0
>iface eth0 inet static
>    address 192.168.1.10
>    netmask 255.255.255.0
>```


- **Vérification :** `ip a`

### Changer la passerelle par défaut (Gateway)

**Ce qu'il faut faire :** Définir la route de sortie pour que la VM puisse communiquer avec l'extérieur ou d'autres réseaux.

- **Sur Debian (/etc/network/interfaces) :** Éditer le fichier.

>[!Changement]
>
>```
># Lignes déjà présentes :
>source /etc/network/interfaces.d/*
>auto lo
>iface lo inet loopback
>
>auto eth0
>iface eth0 inet static
>    address 192.168.1.10
>    netmask 255.255.255.0
># Ajout de la passerlle par defaut
>    gateway 192.168.1.254
># Ajout du DNS 
>    dns-nameservers 8.8.8.8 1.1.1.1
>```

### Configurer le serveur DNS

**Ce qu'il faut faire :** Indiquer au système quel serveur interroger pour résoudre les noms de domaine (ex: google.fr).

- **Action :** Éditer le fichier `/etc/resolv.conf` ou le configurer via Netplan.
- **Configuration :** Ajouter `nameserver 8.8.8.8` ou l'IP de ton serveur DNS local (Bind/Technitium).

### Vérifier la connectivité entre deux VMs

**Ce qu'il faut faire :** Tester si le "tuyau" réseau fonctionne entre deux machines.

- **Commande :** `ping [IP_DE_L_AUTRE_VM]`
