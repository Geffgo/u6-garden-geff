# 2.5 DNS Technitium

### Lancer le conteneur Technitium

- **Ce qu'il faut faire :** Utilise Docker Compose pour déployer le serveur DNS de manière isolée. Assure-toi d'avoir ton fichier `docker-compose.yml` prêt dans le dossier.
- **Commande :** `docker compose up -d`

> [!TIP]
> 
> **Conseil :** `docker images` pour voir les images UP ou DOWN

### Accéder à l'interface web

**Ce qu'il faut faire :** Ouvre ton navigateur pour configurer le serveur via son interface d'administration (port par défaut 5380).

- **URL :** `http://[IP GLPI]:5380`
- **Résultat attendu :** Affichage du tableau de bord Technitium (il te sera demandé de créer un compte admin à la première connexion).

> [!COMPTE]
> 
> - **Admin** : `admin` / `netlab123`

### Créer une zone DNS

- **Ce qu'il faut faire :** Définir le domaine dont ton serveur sera le "maître" (ex: `lab.local`).
- **Menu :** `Zones > Add Zone`
- **Action :** Entre le nom de ta zone (ex: `monprojet.lan`) et valide.

### Créer un enregistrement A

- **Ce qu'il faut faire :** Faire correspondre un nom de machine à une adresse IP (ex: `glpi.monprojet.lan` -> `172.16.1.10`).
- **Menu :** Clique sur ta zone > `Add Record`
- **Action :** Type: `A`, Name: `glpi`, IP Address: `[TON_IP]`.
- **Résultat attendu :** L'enregistrement est ajouté à la table DNS.

### Tester la résolution DNS

- **Ce qu'il faut faire :** Interroger directement ton serveur Technitium pour voir s'il connaît l'adresse que tu viens de créer.
- **Commande :** `nslookup glpi.monprojet.lan 127.0.0.1` (ou l'IP du serveur)
- **Résultat attendu :** Le serveur répond avec l'adresse IP correspondante que tu as configurée.

### Arrêter le conteneur

- **Ce qu'il faut faire :** Une fois les tests terminés, on nettoie les ressources Docker.
- **Commande :** `docker compose down`
