# 2.3 Routeur Cisco 2901

### Réinitialiser le routeur (Remise à zéro)

**Ce qu'il faut faire :** Effacer la configuration de démarrage (`startup-config`) pour repartir de zéro.

- **Commandes :** 1. `write erase` (Valider avec Entrée) 2. `reload` (Confirmer le redémarrage sans sauvegarder)
- **Résultat attendu :** Le routeur redémarre en mode "Initial Configuration Dialog" (répondre `no`).

### Créer une sous-interface (Encapsulation 802.1Q)

**Ce qu'il faut faire :** Diviser une interface physique (ex: GigabitEthernet 0/0) en plusieurs interfaces virtuelles pour gérer les différents VLANs provenant du Switch (Trunk).

- **Commandes :**
    ```
    conf t
    interface gi0/0.10         # .10 correspond souvent au numéro du VLAN
    encapsulation dot1Q 10     # Indique au routeur de lire les tags VLAN 10
    exit
    ```
- **Action :** La sous-interface est créée et prête à recevoir une IP.

### Adresser une sous-interface

**Ce qu'il faut faire :** Assigner une adresse IP qui servira de **passerelle (gateway)** pour les machines du VLAN concerné.

- **Commandes :**
    ```
    interface gi0/0.10
    ip address 192.168.10.254 255.255.255.0
    no shutdown                # Activer l'interface physique parente si ce n'est pas fait
    ```
- **Vérification :** `show ip interface brief` (Le statut doit être `up / up`).

### Configurer ip helper-address (DHCP Relay)

**Ce qu'il faut faire :** Permettre aux requêtes DHCP des clients (VLAN 20) de traverser le routeur pour atteindre un serveur DHCP situé dans un autre réseau (ex: VLAN 10 ou DMZ).

- **Commandes :**
    ```
    interface gi0/0.20         # Sur l'interface où se trouvent les CLIENTS
    ip helper-address 192.168.02.30 # Adresse IP du serveur DHCP distant
    ```
- **Résultat attendu :** Un client dans le VLAN 10 reçoit une adresse IP du serveur DHCP distant.

### Vérifier le routage inter-VLAN

**Ce qu'il faut faire :** Tester la communication entre deux machines situées dans des VLANs différents pour confirmer que le routeur fait bien son travail.

- **Test :** `ping [IP_MACHINE_VLAN_20]` (depuis une machine du VLAN 10).
- **Résultat attendu :** Réponses ICMP reçues. Si échec, vérifie que les passerelles par défaut sur les machines pointent bien vers l'IP des sous-interfaces du routeu
