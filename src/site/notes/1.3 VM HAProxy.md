# 1.1 VM Serveur web 2

> [!ATTENTION]
> La VM abrite plusieurs services vérifier que tout les autres services sont désactiver:
> - DHCP
> - DNS (si pas besoin)
> - 
> Activer le service `HAProxy`avec la commande `systemctl start haproxy.service`


### 1. Informations de la VM
- **Nom de la VM** : `VM-SERVEUR`
- **OS** : Linux Debian 12 (64-bit)
- **Accès Système** :
    - **Utilisateur** : `sysadmin`        
    - **Mot de passe** : `netlab123`
- **Accès Web GLPI (par défaut)** : `glpi` / `glpi`

### Commandes d'administration HAProxy

|**Catégorie**|**Commande**|**Description**|
|---|---|---|
|**Gestion du service**|`systemctl start haproxy`|Démarrer le service.|
||`systemctl stop haproxy`|Arrêter le service.|
||`systemctl restart haproxy`|Redémarrer (coupe les connexions en cours).|
||`systemctl reload haproxy`|**Recharger** la config sans coupure de service.|
||`systemctl status haproxy`|Vérifier l'état et voir les dernières erreurs.|
|**Configuration**|`haproxy -c -f /etc/haproxy/haproxy.cfg`|**Vérifier la syntaxe** du fichier avant reload.|
||`nano /etc/haproxy/haproxy.cfg`|Modifier le fichier de configuration principal.|
|**Diagnostic**|`haproxy -v`|Afficher la version installée.|
||`haproxy -vv`|Afficher la version et les options de compilation.|
||`journalctl -u haproxy -f`|Consulter les logs en temps réel.|
|**Réseau**|`ss -tlnp|grep haproxy`|
||`haproxy -db -f /etc/haproxy/haproxy.cfg`|Lancer en mode **debug** (affiche tout en console).|
