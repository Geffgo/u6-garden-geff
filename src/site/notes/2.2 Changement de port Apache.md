# 2.2 Changement de port Apache

### Modifier le port d'écoute dans `ports.conf`


- **Ce qu'il faut faire :** Ouvre le fichier avec `nano`. Cherche la ligne `Listen 80`. Remplace `80` par ton nouveau port (par exemple `8080`).
- **Commande :** `vim /etc/apache2/ports.conf`
- **Action :** Change `Listen 80` en `Listen 8080`. 

```
# If you just change the port or add more ports here, you will likely also 
# have to change the VirtualHost statement in 
# /etc/apache2/sites-enabled/000-default.conf 

Listen 80 

<IfModule ssl_module> 
	Listen 443 
</IfModule> 

<IfModule mod_gnutls.c> 
	Listen 443 
</IfModule>
```

> [!Changement]
> 
> `Listen [PORT D'ECOUTE]`

#### Modifier le VirtualHost pour le nouveau port

Même si Apache "écoute" le port, il faut dire au site web GLPI de répondre sur ce port spécifique.

- **Ce qu'il faut faire :** Ouvre ton fichier de configuration GLPI. À la toute première ligne, tu verras `<VirtualHost *:80>`. Remplace le `80` par ton nouveau port.
- **Commande :** `nano /etc/apache2/sites-enabled/glpi.conf`
- **Action :** Change `<VirtualHost *:80>` en `<VirtualHost *:8080>`.

```
<VirtualHost *:80>
    DocumentRoot /var/www/glpi/public
    <Directory /var/www/glpi/public>
        Require all granted
        RewriteEngine On
        RewriteCond %{HTTP:Authorization} ^(.+)$
        RewriteRule .* - [E=HTTP_AUTHORIZATION:%{HTTP:Authorization}]
        RewriteCond %{REQUEST_FILENAME} !-f
        RewriteRule ^(.*)$ index.php [QSA,L]
    </Directory>
</VirtualHost>
```

> [!Changement]
> 
> `<VirtualHost *:[PORT D'ECOUTE]>`
### Redémarrer Apache sans erreur

Pour que les modifications soient prises en compte, il faut relancer le moteur d'Apache.

- **Ce qu'il faut faire :** Lance la commande de redémarrage. Si rien ne s'affiche, c'est que c'est bon. Si tu as un message d'erreur, vérifie que tu n'as pas fait de faute de frappe dans les fichiers précédents.
- **Commande :** `systemctl restart apache2`

### Vérifier que le nouveau port est en écoute

On demande au système de nous montrer tous les "tuyaux" (ports) ouverts pour être sûr qu'Apache utilise le bon.

- **Ce qu'il faut faire :** Tape la commande. Tu dois voir apparaître une ligne contenant `*:8080` (ou ton port) et le mot `apache`.
- **Commande :** `ss -tlnp | grep apache`

### Accéder à GLPI sur le nouveau port

On teste si le site répond bien sur sa nouvelle adresse locale.

- **Ce qu'il faut faire :** Utilise `curl` pour "appeler" le site. Si tu vois du code s'afficher (avec des balises HTML), c'est que le site fonctionne sur ce port.
- **Commande :** `curl http://localhost:[Nouveau PORT]`

### Vérifier que l'ancien port 80 ne répond plus

C'est la preuve finale : le port 80 doit être totalement fermé pour Apache.

- **Ce qu'il faut faire :** Tente d'appeler l'ancien port. Le système doit te répondre immédiatement que la connexion est impossible.
- **Commande :** `curl http://localhost:[Ancien PORT]
- **Résultat attendu :** `Failed to connect` ou `Connection refused`.
