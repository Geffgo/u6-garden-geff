# 2.3 Changement de port SSH

### Modifier le port d'écoute dans sshd_config

- **Ce qu'il faut faire :** Ouvre le fichier de configuration du démon SSH. Cherche la ligne `#Port 22`. Retire le symbole `#` (qui sert à commenter) et remplace `22` par ton nouveau port (par exemple `2222`).
- **Commande :** `nano /etc/ssh/sshd_config`
- **Action :** Change `#Port 22` en `Port 2222`.

```
[...]
# The strategy used for options in the default sshd_config shipped with 
# OpenSSH is to specify options with their default value where 
# possible, but leave them commented. Uncommented options override the 
# default value. 

Include /etc/ssh/sshd_config.d/*.conf 

#Port 22 
#AddressFamily any 
#ListenAddress 0.0.0.0 
#ListenAddress ::
[...]
```

> [!Changement]
> 
> `#Port 22 ` -> `Port [Nouveau PORT] `
### Redémarrer le service SSH

- **Ce qu'il faut faire :** Applique la configuration en relançant le service. Contrairement à d'autres services, redémarrer SSH ne coupe généralement pas ta session actuelle, mais reste prudent.
- **Commande :** `systemctl restart ssh` (ou `sshd`)

### Vérifier que le nouveau port est en écoute

- **Ce qu'il faut faire :** On vérifie que le système écoute bien sur le nouveau port défini.
- **Commande :** `ss -tlnp | grep sshd`

### Se connecter en SSH sur le nouveau port

- **Ce qu'il faut faire :** Teste la connexion depuis une autre machine (ou en ouvrant un nouveau terminal) en précisant explicitement le port avec l'option `-p`.
- **Commande :** `ssh -p [NOUVEAU_PORT] utilisateur@adresse_ip`

### Vérifier que l'ancien port 22 ne répond plus

- **Ce qu'il faut faire :** On tente de se connecter sur le port par défaut pour confirmer qu'il est bien fermé.
- **Commande :** `ssh -p 22 utilisateur@adresse_ip`
